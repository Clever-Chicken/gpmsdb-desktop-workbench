# GPMsDB 桌面版使用说明

这份文档面向实际使用者，说明如何准备数据库、在桌面端导入原始数据库目录或加载现成数据库、选择查询文件、查看结果以及导出数据库 CSV。

## 1. 环境准备

建议在项目根目录执行以下命令：

```bash
corepack pnpm --dir apps/desktop install
cargo test -p gpmsdb-format
cargo test -p gpmsdb-desktop-tauri
corepack pnpm --dir apps/desktop test
```

如果测试全部通过，再启动桌面端：

```bash
corepack pnpm --dir apps/desktop tauri dev
```

## 2. 准备数据库

桌面端读取的是已经构建好的 mmap 数据库目录，目录中至少应包含：

- `header.bin`
- `mass_index.bin`
- `genome_peaks.bin`
- `meta.bin`

如果你只有原始 Pickle 数据，现在有两种方式：

### 方式 A：直接在桌面端导入原始数据库目录

启动桌面端后，点击“导入原始数据库目录并构建”：

1. 选择原始发布包目录，例如 `R01-RS95`，也可以选择它下面的 `mass/` 或 `custom/` 等相关目录
2. 再选择一个输出目录
3. 程序会调用内置 Rust builder 构建 `header.bin / mass_index.bin / genome_peaks.bin / meta.bin`
4. 构建成功后会自动把该输出目录作为当前数据库加载

这一步对目录名不敏感。也就是说，就算不是 `R01-RS95` 这个名字，只要目录层级附近能找到可用的 `all.db` 及配套元数据，依然可以导入。

### 方式 B：在终端先运行 builder

如果你更喜欢命令行，也可以先运行 builder：

```bash
cargo run -p gpmsdb-builder -- \
  --source-root tests/fixtures/small_source \
  --out-dir .tmp/small-db \
  --progress jsonl
```

构建完成后，把 `.tmp/small-db` 当作数据库目录即可。

## 3. 启动桌面端后的操作步骤

### 第一步：选择数据库目录或先导入原始数据库目录

点击“选择数据库目录”，选中包含四个 `.bin` 文件的数据库目录。

如果你手里只有原始发布包，则改为点击“导入原始数据库目录并构建”，先选中原始数据库目录，再选输出目录。

成功后，界面会显示：

- 已加载数据库：`<你的目录>`

同时日志面板也会记录这一步。

### 第二步：选择查询文件

点击“选择查询文件”，可多选本地质谱文件。

当前支持：

- `.mgf`
- `.txt`

界面会显示已选择的查询文件数量。

### 第三步：开始批量鉴定

点击“开始批量鉴定”后：

- 顶部进度文本会变化
- 日志会显示任务已启动
- 结果表会在后端完成计算后显示识别结果

## 4. 查询文件格式要求

当前版本支持作者工具常见的文本峰列表格式，最稳妥的输入格式是每行两列：

```txt
1000.0 0.50
1200.0 0.30
1001.0 0.20
```

也支持只有一列 `m/z` 的行，此时强度会默认按 `1.0` 处理。例如：

```txt
COM=sample-a
# comment
1000.0
1200.0 7.5
```

对于 `.mgf` 文件，当前实现会跳过大多数元信息行，只读取能解析成：

```txt
m/z intensity
```

形式的峰值行。

如果文件中没有任何可解析峰值，程序会报错。

## 5. 结果表怎么看

结果表中的列含义如下：

- `查询文件`
  - 来自哪个输入文件
- `基因组 ID`
  - 命中的数据库基因组编号
- `得分`
  - 重排序后的分数
- `匹配峰数`
  - `matched_ribosomal / matched_total`

如果后续需要更详细结果，可以继续扩展为：

- 基因组名称
- 分类信息
- 候选排序
- 导出结果 CSV

## 6. 导出数据库 CSV

点击“导出数据库 CSV”后，选择一个本地保存路径，程序会导出包含以下字段的 CSV：

- `genome_id`
- `display_name`
- `taxonomy`
- `total_peaks`
- `gene_count`

这适合做：

- 数据检查
- 课程展示
- 后续 Excel 分析

## 7. 常见问题

### 7.1 点击开始批量鉴定没有结果

请先检查：

- 是否已经加载数据库
- 是否已选择查询文件
- 查询文件中是否真的包含可解析的峰值行

### 7.2 查询文件支持 mzML 吗？

当前还不支持。现在实现的是一个最小可运行骨架，后续可以加入更完整的质谱文件解析器。

### 7.3 可以取消任务吗？

当前“取消”按钮仍是占位，尚未连通后端取消逻辑。

## 8. 推荐演示流程

课堂演示时建议按下面顺序操作：

1. 启动 `tauri dev`。
2. 如果需要，从 GUI 里导入原始数据库目录并构建数据库。
3. 或者直接选择已经构建好的数据库目录。
4. 选择两到三个示例查询文件。
5. 点击“开始批量鉴定”。
6. 展示进度变化、结果表格和日志。
7. 最后点击“导出数据库 CSV”，展示导出能力。
